# core-schemas-ts

**Tier 0 — Foundation.**

TypeScript twin of `core-domain-contracts`. Generated types from the canonical JSON Schemas.

## Source

Generated by `scripts/codegen.ts` (or similar) which reads the JSON Schemas published by `packages/py/core-domain-contracts/` and emits TS types.

## Responsibilities

- Type-only package: `interface`, `type`, and enum exports for every domain entity.
- No runtime code.
- Contract round-trip test: a JSON fixture is parsed by Pydantic in CI and the resulting JSON is compared to the same fixture parsed by the generated TS types — drift fails CI.

## Dependencies

- External: none (or `zod` if we choose runtime validation — TBD)
- Internal: none

## Consumers

Every TS package and app shell.
