# core-domain-contracts

**Tier 0 — Foundation. No internal deps.**

Canonical Pydantic v2 types and JSON Schemas for every domain entity that crosses package boundaries: documents, runs, references, users, tenants, teams, access groups, plus reusable activity/measurement schemas.

## Source material

Merges:
- `Process Wizard Docs/libraries/pw-domain-contracts.md` (runs, documents, workflows, references, enums)
- `Organizer/shared-schemas.md` (activity payloads, measurement schemas in `schemas/activity/v1/` and `schemas/user/v1/`)

## Responsibilities

- Pydantic v2 models for cross-package contracts.
- JSON Schemas as the **single source of truth** for cross-language types (TS twin generated by `core-schemas-ts`).
- Enum definitions used by API/service/repository code.
- Versioned schema directories (`schemas/<domain>/v1/`).

## Dependencies

- External: `pydantic`
- Internal: none

## Consumers

Everything. This is the lowest layer.

## Notes

- Keep this package framework-free (no FastAPI, no SQLAlchemy imports).
- Schema changes require a contract round-trip test fixture update in `core-schemas-ts`.
